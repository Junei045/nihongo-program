このプログラムの裏側（第3回ワークショップ）

みなさんが書いた「日本語の命令」は、どうやってコンピュータに伝わるのでしょう？　「のぞき窓」で見えていた英語が、実際の仕組みです。

ステップ1：字句解析（じくかいせき）― 命令を「単語」に分ける

まず、1行の命令を「単語のかたまり」に分解します。この作業を トークナイズ といいます。

日本語命令：「ねんれい」に 10 を いれる ↓ 分解すると… [ 変数:"ねんれい" ][ キーワード:"に" ][ 数字:10 ][ キーワード:"を" ][ キーワード:"いれる" ]
人間は「ねんれいに10をいれる」を一気に読めますが、コンピュータは1文字ずつしか読めません。だから「どこで区切るか」を最初に決める必要があるのです。

実際のJavaScriptコード（このページの中に入っています）：

function tokenize(line) { // 「」で囲まれた部分 → 変数名 if (line[i] === '「') { let j = line.indexOf('」', i+1); tokens.push({ type: 'var', val: line.slice(i+1, j) }); } // 数字 → 数値 else if (/[0-9]/.test(line[i])) { // 数字が続く間つなげる while (/[0-9.]/.test(line[i])) num += line[i++]; tokens.push({ type: 'num', val: parseFloat(num) }); } // それ以外 → キーワード else { // 空白まで読む tokens.push({ type: 'kw', val: word }); } }

ステップ2：実行（じっこう）― 命令を動かす

分解した単語を見て「どの命令か」を判断し、実際に動かします。これを インタープリタ（翻訳係）といいます。

// 「〜」と表示する → の場合 if (tok[0].type === 'var' && tok[1].val === 'と表示する') { output.push(tok[0].val); // 出力に追加する } // N かいくりかえす → の場合 if (tok[0].type === 'num' && kw.includes('かいくりかえす')) { const times = tok[0].val; // 回数 for (let k = 0; k < times; k++) { // その回数だけ execLines(body, output); // 中身を実行 } }
「もし〜ならば」「くりかえし」は、再帰（さいき）という方法を使っています。命令の中に命令があっても、同じ関数を何度も呼ぶことで処理できます。

ステップ3：かめ描画の仕組み

かめの絵は Canvas（キャンバス） という機能を使っています。かめの位置（x, y）と向き（角度）を変数で管理して、「すすむ」たびに線を引いています。

let tx = 350; // かめのX位置（横） let ty = 140; // かめのY位置（縦） let tangle = -90; // かめの向き（上向きからスタート） function tMove(d) { const rad = tangle * Math.PI / 180; // 角度をラジアンに変換 const nx = tx + d * Math.cos(rad); // 新しいX const ny = ty + d * Math.sin(rad); // 新しいY ctx.beginPath(); ctx.moveTo(tx, ty); // 今の場所から ctx.lineTo(nx, ny); // 新しい場所へ線を引く ctx.stroke(); tx = nx; ty = ny; // 位置を更新 }

「90 まがる」は角度を90度増やすだけ。「4かいくりかえす → 80すすむ → 90まがる」で正方形が描けるのは、4回 × 90度 = 360度（一周）になるからです！
